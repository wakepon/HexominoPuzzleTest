# 加護（Blessing）システム — 第4のエフェクト層

## ステータス: 導入推奨

ブロックに付与された「加護」が、消去後にセルに永続効果として残る仕組み。
シールと同じ入り口（ピースに付与）から、寿命が真逆の効果を生む。

---

## コンセプト

### 基本フロー
```
1. ショップでピースを購入 → 加護がブロックに付いている（シールと同列）
2. ピースを盤面に配置
3. ライン消去でブロックが消える
4. 消えたセルに「加護」が永続的に残る ← ここがシールとの違い
5. 以後、そのセルに配置されたブロックが加護の恩恵を受ける
```

### シールとの対比

| | シール | 加護 |
|--|--------|------|
| 付与先 | ブロック1個 | ブロック1個 → 消去後セルに残る |
| 入手方法 | ピースに付与済み（同じ） | ピースに付与済み（同じ） |
| 寿命 | ブロックと一緒に消える | 消去で誕生 → ラン永続 |
| 効果発動 | 消去時に即時効果 | そのセルに置かれた将来のブロックに効果 |
| 性質 | **消費型**（使い捨て） | **投資型**（永続リターン） |
| 効果対象 | **ブロック**に作用 | **セル**に作用 |

→ 同じ「ブロックに付与」の入り口から、寿命が真逆の2種類の効果が生まれる。

### シールと加護の併存

**1ブロックにシールと加護の両方が付与可能。**

別カテゴリに分ける理由:
- シール = ブロックへの効果（ブロックが消えると消失）
- 加護 = セルへの効果（ブロックが消えた後に残る）
- **対象が異なる（ブロック vs セル）** ので、両方付いていても概念的に矛盾しない
- 統一すると「このシールは消える」「このシールは残る」が混在し、プレイヤーが混乱する
- 排他にすると「なぜ同時に付けられないのか」に自然な理由がない（ゲーム都合に見える）

例: 「multiシール + 力の加護」のブロック
→ 消去時: multiでスコア2倍（シール効果、消費）+ セルに力の加護が残る（永続投資）
→ 一つの消去で二重の恩恵。「お得」な当たりブロック。

### ピースの付与構成

```
ピース = パターン(0-1) + シール(ピース中0-1ブロック) + 加護(ピース中0-1ブロック)
```

※ シールと加護は同じブロックに付くことも、別ブロックに付くこともある

| ピース構成 | 体感 |
|-----------|------|
| 素のピース | 普通 |
| パターンのみ | ちょっと嬉しい |
| シールのみ | 嬉しい |
| 加護のみ | 嬉しい |
| パターン + シール | かなり嬉しい |
| パターン + 加護 | かなり嬉しい |
| シール + 加護 | レア |
| パターン + シール + 加護 | 大当たり |

→ 「大当たり」ピースの存在がショップの楽しさ（ガチャ的快感）を生む。

## エフェクト4層構造（最終形）

```
レリック（永続・プレイヤー）  ← シナジーの中心
パターン（一時・ピース単位）  ← 戦術の幅
シール  （一時・ブロック単位）← 即時の希少効果
加護    （永続・セル単位）    ← 盤面育成の投資
```

| 層 | 付与先 | 付与元 | 寿命 | 効果タイミング |
|----|--------|--------|------|-------------|
| レリック | プレイヤー | ショップ直接購入 | ラン永続 | 自動 |
| パターン | ピース全体 | ピースに付与済み | 配置〜消去 | 配置時/消去時 |
| シール | ブロック1個 | ピースに付与済み | 配置〜消去 | 消去時 |
| **加護** | **ブロック1個→セル** | **ピースに付与済み** | **消去で誕生→ラン永続** | **そのセルで次回以降の消去時** |

**対称構造:**

|  | 永続（投資型） | 一時（消費型） |
|--|---------------|-------------|
| **プレイヤー / ピース** | レリック | パターン |
| **セル / ブロック** | 加護 | シール |

## なぜこの設計が優れているか

### 1. 追加UIがゼロ

- ショップに新枠不要（既存のピース枠にシールと並列で付与）
- 配置先の選択UI不要（ピースの配置 = 加護の配置先決定）
- プレイヤーは「ピースを置く」という既存操作だけでフィールドを育てられる

### 2. 一つの操作に3つの判断軸

ピース配置時の判断:
1. 「どこに置けばラインが完成するか」（既存）
2. 「消去後にどのセルに加護が残ると嬉しいか」← NEW
3. 「既存の加護セルの上にブロックを置いて恩恵を受けるか」← NEW

**操作の複雑度は変わらないのに、戦略の深みだけが増す。**

### 3. 消去に「投資」の意味が生まれる

- 現状: ライン消去 = スコア獲得
- 加護後: ライン消去 = スコア獲得 + フィールド育成

「消す」ことが「育てる」ことになる。パズルゲームとして美しい循環。

### 4. ボスのobstacleとの緊張感

- 加護で育てたセルにobstacleが乗ると効果が無効化される
- obstacleは消去不可 → 事実上「加護が潰される」
- 「加護育成一辺倒」だとボス戦で壊されるリスク
- フィールドに投資しすぎる → ゴールドがレリック/ピースに回らない → デッキが弱い

→ レリック投資 vs ピース(パターン/シール)投資 vs フィールド(加護)投資の3方向バランス

### 5. aura削除の穴を完全に埋める

前回のリストラでauraを削除 → 「配置位置を考える」メカニクスが減少。
加護が盤面にあることで「どこに置くか」の判断軸が自然に復活する。

## 具体的な設計案

### 付与ルール

- ピースに「パターン」「シール」「加護」がそれぞれ独立して付与される（併存可能）
- シール: ピース中の0-1ブロックに付与
- 加護: ピース中の0-1ブロックに付与（シールと同じブロックでも可）
- 加護付きピースの出現率はシールと同程度 or やや低め

### 加護の重ね掛け（レベルシステム）

**同じセルに同種の加護を重ねるとレベルアップする。**

| レベル | 効果倍率 | 到達に必要な回数 |
|--------|---------|---------------|
| Lv1 | ×1（基本） | 1回 |
| Lv2 | ×2 | 2回（or multi1回） |
| Lv3 | ×3（上限） | 3回 |

**ルール:**
- 1セルに付けられる加護は1種類のみ
- 同種の加護 → レベルアップ（Lv3が上限）
- 異種の加護 → 付かない（上書きされない）
- → 「このセルをどの加護で育てるか」を最初に決めるコミットメントが生まれる

**multiシールとのシナジー:**
- multiシール付き加護ブロックが消去される → 加護が2段階分適用（Lv0→Lv2、Lv1→Lv3）
- 「multi + 加護」の大当たりブロックで一気にLv2到達

**集中 vs 分散の投資判断:**

| 戦略 | メリット | リスク |
|------|---------|-------|
| **集中**（1セルをLv3に） | 高リターン（+3） | obstacle1つで大損失 |
| **分散**（3セルをLv1ずつ） | リスク分散 | 個々の効果は+1 |
| **バランス型** | 中間 | 中間 |

→ Slay the Spireの「カードをアップグレードするか、新しいカードを追加するか」と同じ構造。
  デッキ構築ゲームの核心的な面白さ。

### 加護の効果案（初期3〜5種）

加護は永続かつレベルアップ可能なので、Lv1の効果は**控えめ**に。

| 名前 | Lv1 | Lv2 | Lv3 | 方向性 |
|------|-----|-----|-----|--------|
| **力の加護** | ブロック点+1 | +2 | +3 | スコア加算（基本） |
| **金の加護** | 消去時+1G | +2G | +3G | 経済系 |
| **連の加護** | ライン点+1 | +2 | +3 | ライン強化 |
| **導の加護** | パターン効果+1段階 | +2段階 | +3段階 | シナジー系 |
| **癒の加護** | 消去時に手数+1 | +1(確率UP?) | +1(確定?) | リソース回復（希少） |

※ 癒の加護は手数回復が強力なため、Lv1でも確率制にする等のバランス調整が必要かもしれない

### 視覚表現

- **空きセル + 加護**: セル背景に淡いシンボル。Lvに応じて光が強くなる
  - Lv1: 淡い紋様
  - Lv2: やや明るい紋様
  - Lv3: はっきりした紋様 + 微かな光のエフェクト
- **ブロックが乗っている + 加護**: セルの縁取りが光る（Lvで明るさ変化）
- **obstacleが乗っている + 加護**: 加護マーカーが暗く表示（封印状態）
- ブロックへの付与時: シールと同様にシンボル表示（色味やデザインで区別）
- Lv表示: セル内に小さく数字 or シンボルの大きさで表現

### 永続性ルール

- ラン中永続（ゲームオーバーでリセット）
- ブロックの配置/消去では加護は消えない
- obstacleが上に乗ると加護効果は無効化（obstacleが除去されれば復活）
- 1セル1種の加護（同種のみレベルアップ、異種は無効）
- Lv3が上限

## 命名: 「加護」

### 採用理由
- ポジティブな響き（守護、祝福のニュアンス）
- 「ブロックが消えた後に、そのセルが祝福される」という物語性
- 短い（2文字）でUI表示に適している
- RPGゲーム用語として馴染みがある

### 英語名候補
- **Blessing** — 最も直訳的で自然
- Grace, Boon, Ward も候補

### 各加護の命名パターン
「〇の加護」で統一 → 力の加護、金の加護、連の加護...
シンプルで一貫性がある。

---

## ボスのobstacleとの相互作用（数値分析）

- obstacle: ランダムな4セルに配置
- 6×6 = 36セル

| 加護セル数 | 被害確率 | 備考 |
|-----------|---------|------|
| 1セル（Lv3集中） | ≈ 11% | 低確率だが被害時の損失が大きい |
| 3セル（Lv1分散） | ≈ 30% | 適度なリスク |
| 6セル | ≈ 52% | 半分以上の確率で被害 |

→ **集中投資はobstacle耐性が高いが、当たった時の損失が大きい（Lv3消失）。**
→ **分散投資はobstacleに当たりやすいが、1セル分のLv1損失で済む。**
→ リスクの性質が異なるため、どちらが正解とも言い切れない。これが面白さ。

## 感情曲線

1. **序盤**: 最初の加護ブロックを消去 → 「セルが光った！」→ 発見の喜び
2. **中盤**: 3〜5セルに加護が広がる → 「自分の盤面が育ってる」→ 成長の実感
3. **ボス戦**: obstacleが加護セルの近くに... → 緊張と祈り
4. **ボス後**: 被害確認 → 再建 or 新規育成の判断
5. **終盤**: 育った加護 + レリック + パターンのフルシナジー → カタルシス

## 段階的導入の推奨

1. **Phase 1**: 基本2種（力の加護・金の加護）で仕組みを実装
2. **Phase 2**: 残り3種追加 + 加護対応レリック
3. **Phase 3**: 加護とパターン/シールの高度なシナジー設計
